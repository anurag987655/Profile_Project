# Reinforcement Learning from Human Feedback

RLHF is the secret sauce behind modern conversational AI. The pipeline has three stages:

## 1. Supervised Fine-Tuning (SFT)

Start with a pre-trained base model and fine-tune it on high-quality demonstrations of desired behaviour — typically human-written prompt/response pairs. This teaches the model the basic style and format we want.

## 2. Reward Model Training

Collect a dataset of model outputs ranked by humans (e.g., "response A is better than response B"). Train a separate reward model to predict the human preference score. This model serves as a proxy for the objective we actually care about.

## 3. Proximal Policy Optimisation (PPO)

Use the reward model to provide feedback to the language model. PPO updates the policy (the LM) to maximise expected reward while staying close to the SFT model via a KL penalty — preventing the model from exploiting the reward function by generating nonsense that happens to score highly.

A key insight: the reward model is imperfect, so the KL constraint is not just a nice-to-have but a necessity. Without it, the model quickly diverges into high-reward gibberish. The PPO clip ratio and KL coefficient are the most sensitive hyperparameters in the entire pipeline.

---

### Practical Considerations

| Parameter | Typical Range | Sensitivity |
|-----------|--------------|-------------|
| KL coefficient | 0.01 – 0.5 | High |
| PPO clip ratio | 0.1 – 0.3 | High |
| Learning rate | 1e-6 – 5e-6 | Medium |
| Batch size | 32 – 128 | Low |
